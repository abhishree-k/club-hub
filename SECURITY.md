# Security Policy

DSCE Clubs (Club Hub) is a centralized platform for club management, event coordination, and campus communication. Since the system may handle user accounts and institutional information, security and privacy are taken seriously.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

⚠️ **Do not open a public GitHub issue for security vulnerabilities.**  
Public reports may expose sensitive information or platform weaknesses.

### How to Report
Please report security issues privately by [contacting the maintainers](README.md#connect-with-us)

When reporting, include:
- A clear description of the issue
- Steps to reproduce (proof-of-concept if possible)
- Affected pages/modules/components
- Expected vs actual behavior
- Potential impact (e.g., unauthorized access, data exposure)

---

## Scope

Security vulnerabilities may include (but are not limited to):
- Authentication and authorization issues (login/session handling)
- Broken access control (students/club leads/admin privileges)
- Data leaks involving student or event data
- Injection attacks (SQL injection, XSS, etc.)
- Insecure file uploads (posters, documents, images)
- Exposure of secrets (API keys, database credentials)

---

## Responsible Disclosure

Please allow maintainers reasonable time to investigate and resolve the issue before any public disclosure.

Thank you for helping keep DSCE Clubs secure and reliable.
